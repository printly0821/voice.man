## 1. 한국 형사소송법 Article 313(2)(3)

### 1.1 법률 개요

**형사소송법 제313조(증거능력의 제한)**

제2항: 전자문서는 진정성립이 인정된 경우에 한하여 증거로 사용할 수 있다.

제3항: 전자문서의 진정성립은 그 전자문서가 특정인에 의하여 작성 또는 송부되었다는 사실 및 그 내용이 작성 시점부터 제출 시점까지 변경되지 아니하였다는 사실을 증명함으로써 인정된다.

### 1.2 준수 요구사항

#### 1.2.1 디지털 증거 무결성 입증

- [ ] **SHA-256 해시 생성 및 기록**
  - 증거 수집 즉시 SHA-256 해시 생성
  - 해시 값을 별도 문서에 기록
  - 제출 시점까지 해시 값 일치 확인
  - **구현 위치**: `src/voice_man/forensics/evidence/digital_signature.py`
  - **검증 방법**: 파일 업로드 시 자동 해시 생성 및 비교

- [ ] **전자서명 적용**
  - RSA 2048-bit 전자서명 생성
  - Private Key로 해시 서명
  - Public Key 보고서에 첨부
  - **구현 위치**: `src/voice_man/forensics/evidence/digital_signature.py`
  - **검증 방법**: Public Key로 서명 검증

- [ ] **타임스탬프 기록**
  - RFC 3161 준수 타임스탬프 발급
  - 수집 시점 기록
  - 타임스탬프 토큰 보관
  - **구현 위치**: `src/voice_man/forensics/evidence/timestamp_service.py`
  - **검증 방법**: TSA 인증서 체인 검증

- [ ] **감사 로그 보관**
  - 모든 접근 이벤트 기록
  - Append-only 로그 사용
  - 해시 체인으로 무결성 보장
  - **구현 위치**: `src/voice_man/forensics/evidence/audit_logger.py`
  - **검증 방법**: 해시 체인 무결성 검증

#### 1.2.2 진정성립 증명

- [ ] **작성자 식별**
  - 분석자 정보 기록 (이름, 자격, 소속)
  - 사용 도구 및 버전 기록
  - 분석 일시 기록
  - **보고서 섹션**: Title Page, 방법론 상세 설명

- [ ] **변경 이력 추적**
  - 초기 수집 시점 기록
  - 모든 이동 및 접근 기록
  - 법정 제출 시점 기록
  - **구현 위치**: Chain of Custody 감사 로그

- [ ] **무결성 검증 절차**
  - 수집 시 해시 vs 제출 시 해시 비교
  - 전자서명 검증
  - 감사 로그 해시 체인 검증
  - **검증 스크립트**: `verify_chain()` 메서드

### 1.3 준수 증명 자료

**법정 제출 시 필수 자료:**

1. **무결성 증명 문서**
   - 초기 SHA-256 해시 기록
   - 현재 SHA-256 해시 확인
   - 해시 일치 확인서

2. **전자서명 검증 문서**
   - Public Key
   - 서명 값
   - 서명 검증 결과

3. **타임스탬프 증명 문서**
   - RFC 3161 토큰
   - TSA 인증서 체인
   - 타임스탬프 검증 결과

4. **감사 로그 요약**
   - 전체 접근 이력
   - 해시 체인 무결성 확인서
   - 주요 이벤트 요약

## 2. ISO/IEC 27037 - 디지털 증거 수집/보존

### 2.1 표준 개요

**ISO/IEC 27037:2012 - Guidelines for identification, collection, acquisition, and preservation of digital evidence**

디지털 증거의 식별, 수집, 획득, 보존에 대한 국제 표준 가이드라인

### 2.2 준수 요구사항

#### 2.2.1 증거 식별 프로토콜

- [ ] **증거 식별**
  - 증거 파일 고유 식별자 부여 (UUID v7)
  - 파일명, 크기, 포맷 기록
  - 메타데이터 수집 (생성 일시, 수정 일시)
  - **구현 위치**: `src/voice_man/models/asset.py`

- [ ] **증거 분류**
  - 증거 유형 분류 (오디오 파일)
  - 사건 번호와 연계
  - 중요도 등급 설정
  - **데이터베이스**: `assets` 테이블

#### 2.2.2 증거 수집 절차

- [ ] **수집 시점 타임스탬프**
  - RFC 3161 타임스탬프 발급
  - 수집 일시 ISO 8601 형식 기록
  - 로컬 시간대(+09:00) 명시
  - **구현 위치**: `src/voice_man/forensics/evidence/timestamp_service.py`

- [ ] **수집자 정보 기록**
  - 수집 담당자 이름, 소속, 연락처
  - 입회자 정보 (선택적)
  - 수집 장소 기록
  - **문서**: 증거 수집 기록지

- [ ] **수집 방법 문서화**
  - 수집 도구 및 방법 기록
  - 수집 환경 기록 (OS, 하드웨어)
  - 수집 과정 사진 촬영 (선택적)
  - **문서**: Chain of Custody 기록

#### 2.2.3 증거 보존

- [ ] **원본 파일과 분석 결과 분리**
  - 원본 파일 읽기 전용 설정
  - 작업 사본 별도 생성
  - 분석 결과 별도 디렉토리 저장
  - **디렉토리 구조**: `data/uploads/`, `data/forensic/`

- [ ] **보존 환경 관리**
  - 안전한 저장소 사용
  - 접근 권한 제한
  - 정기적 백업
  - **보안 설정**: 파일 시스템 권한, 암호화

- [ ] **보존 기간 준수**
  - 법적 보관 기간 확인
  - 보관 기간 만료 전 폐기 금지
  - 폐기 시 승인 절차
  - **정책**: 보관 및 폐기 절차

#### 2.2.4 증거 접근 이력 기록

- [ ] **모든 접근 이벤트 기록**
  - 접근 일시, 접근자, 접근 목적
  - 접근 유형 (읽기, 복사, 이동)
  - 접근 결과 (성공, 실패)
  - **구현 위치**: `src/voice_man/forensics/evidence/audit_logger.py`

- [ ] **변경 이력 추적**
  - 파일 이동 시 이동 전/후 해시 비교
  - 재서명 기록
  - 변경 사유 기록
  - **문서**: 증거 이동 기록지

### 2.3 준수 증명 자료

**ISO/IEC 27037 준수 증명 문서:**

1. **증거 수집 기록지**
   - 수집 일시, 장소, 담당자
   - 증거 식별 정보
   - 초기 해시 값

2. **증거 보존 기록**
   - 보관 위치 및 환경
   - 접근 권한 설정
   - 백업 기록

3. **증거 접근 로그**
   - 전체 감사 로그
   - 주요 접근 이벤트 요약
   - 해시 체인 무결성 확인

## 3. ISO/IEC 17025 - 포렌식 실험실 인정

### 3.1 표준 개요

**ISO/IEC 17025:2017 - General requirements for the competence of testing and calibration laboratories**

시험 및 교정 실험실의 적격성에 대한 일반 요구사항

### 3.2 준수 요구사항

#### 3.2.1 방법론 검증

- [ ] **검증 데이터셋 테스트**
  - Ground Truth 데이터셋 준비
  - 알고리즘 성능 테스트
  - Precision, Recall, F1 점수 평가
  - **구현 위치**: `src/voice_man/forensics/validation/performance_metrics.py`

- [ ] **재현성 테스트**
  - 동일 입력 → 동일 출력 검증
  - Random seed 고정
  - 허용 오차 정의 (±0.0001)
  - **구현 위치**: `src/voice_man/forensics/validation/bootstrap.py`

- [ ] **정확도 메트릭 기록**
  - Precision, Recall, F1 점수
  - Confusion Matrix
  - 신뢰구간 (95%)
  - **구현 위치**: `src/voice_man/forensics/validation/`

- [ ] **방법론 변경 이력**
  - 변경 사유 및 내역 문서화
  - 재검증 수행
  - 이전 버전과 성능 비교
  - **문서**: 방법론 검증 프로토콜

#### 3.2.2 품질 관리 체계

- [ ] **내부 품질 검증**
  - 분석 결과 논리적 일관성 검토
  - 신뢰구간 범위 확인
  - 이상치 탐지 및 조사
  - **구현 위치**: `ForensicScoringService.calculate_forensic_score()`

- [ ] **교차 검증**
  - 다른 분석가의 독립 분석
  - 결과 비교 및 차이 분석
  - 불일치 시 재분석
  - **절차**: 품질 검증 절차

- [ ] **품질 기록 유지**
  - 품질 관리 체계 문서화
  - 검증 결과 기록
  - 개선 조치 사항 기록
  - **문서**: 품질 관리 기록

#### 3.2.3 도구 검증 및 교정

- [ ] **사용 도구 버전 기록**
  - Python 버전, OS 정보
  - WhisperX, PyTorch, KoBERT 버전
  - GPU 정보 (CUDA 버전, GPU 모델)
  - **보고서 섹션**: 사용 도구 명세

- [ ] **라이브러리 체크섬 검증**
  - 설치된 라이브러리 버전 확인
  - SHA-256 체크섬 검증
  - 라이선스 정보 기록
  - **구현**: `pip list`, `pip freeze`

- [ ] **모델 버전 및 출처 기록**
  - AI 모델 버전 (예: whisper-large-v3)
  - 모델 체크섬
  - 훈련 데이터 출처
  - **데이터베이스**: `tool_verification` 테이블

- [ ] **교정 절차**
  - 모델 업데이트 시 성능 비교
  - 이전 버전과 차이 분석
  - 교정 기록 감사 로그 유지
  - **문서**: 도구 검증 및 교정 절차

#### 3.2.4 전문가 자격 요구사항

- [ ] **자격 정보 기록**
  - 학위, 자격증 정보
  - 실무 경험 기록
  - 법정 증언 경험 (선택적)
  - **보고서 섹션**: Title Page

- [ ] **시스템 교육 이수**
  - voice.man 시스템 교육 수료
  - 포렌식 방법론 교육
  - 법적 준수 교육
  - **기록**: 교육 이수 증명서

- [ ] **자격 갱신**
  - 정기적 재교육 (연 1회)
  - 최신 기술 동향 학습
  - 법률 변경 사항 숙지
  - **정책**: 자격 유지 및 갱신

### 3.3 준수 증명 자료

**ISO/IEC 17025 준수 증명 문서:**

1. **방법론 검증 보고서**
   - 검증 데이터셋 설명

   - 재현성 테스트 결과

   - 개선 조치 사항

3. **도구 검증 기록**
   - 사용 도구 버전 및 체크섬
   - 모델 버전 및 출처
   - 교정 기록

4. **전문가 자격 증명**
   - 학위 및 자격증 사본
   - 교육 이수 증명서
   - 경력 증명서

## 4. NIST SP 800-86 - 디지털 포렌식 가이드

### 4.1 표준 개요

**NIST Special Publication 800-86 - Guide to Integrating Forensic Techniques into Incident Response**

디지털 포렌식 기법을 사고 대응에 통합하기 위한 NIST 가이드

### 4.2 준수 요구사항

#### 4.2.1 디지털 포렌식 4단계 절차

**1단계: 수집 (Collection)**

- [ ] **증거 식별 및 수집**
  - 증거 파일 식별
  - 해시 값 생성
  - 타임스탬프 발급
  - **구현**: 증거 수집 단계

- [ ] **수집 도구 및 방법 문서화**
  - 수집 도구 버전 기록
  - 수집 절차 상세 기록
  - 수집 환경 기록
  - **문서**: 증거 수집 기록지

**2단계: 검사 (Examination)**

- [ ] **증거 무결성 검증**
  - 해시 값 일치 확인
  - 전자서명 검증
  - 타임스탬프 검증
  - **구현**: 무결성 검증 스크립트

- [ ] **데이터 추출**
  - 오디오 파일 메타데이터 추출
  - 음성 특성 추출 (음량, 피치, 말속도)
  - 텍스트 변환 (STT)
  - **구현**: WhisperX 파이프라인

**3단계: 분석 (Analysis)**

- [ ] **포렌식 분석 수행**
  - 범죄 언어 패턴 탐지
  - 심리적 조작 패턴 식별
  - 감정 분석
  - **구현**: 포렌식 분석 서비스

- [ ] **통계적 유의성 평가**
  - Bootstrap 95% 신뢰구간 계산
  - 성능 메트릭 산출 (Precision, Recall, F1)
  - 재현성 테스트
  - **구현**: 검증 모듈

**4단계: 보고 (Reporting)**

- [ ] **법정 제출용 보고서 생성**
  - Title Page, Executive Summary
  - 방법론 상세 설명
  - 사용 도구 명세
  - **구현**: HTML/PDF 보고서 생성기

- [ ] **재현 절차 문서화**
  - 단계별 분석 재현 가이드
  - 입력 파일 준비
  - 분석 파라미터 설정
  - **보고서 섹션**: 재현 절차

#### 4.2.2 재현 가능성 보장

- [ ] **분석 파라미터 기록**
  - 모델 버전
  - Random seed
  - 입력 파일 해시
  - **구현**: 분석 메타데이터 기록

- [ ] **동일 입력 → 동일 출력**
  - 재현성 테스트 수행
  - 허용 오차: ±0.0001
  - 차이 발생 시 로그 기록
  - **구현**: `random_seed` 파라미터 고정

- [ ] **재현 절차 문서화**
  - 환경 설정 가이드
  - 실행 명령어
  - 예상 출력 형식
  - **보고서 섹션**: 재현 절차 단계별 설명

#### 4.2.3 완전한 문서화

- [ ] **증거 수집 문서**
  - 증거 수집 기록지
  - 초기 해시 값
  - 타임스탬프 토큰
  - **문서**: Chain of Custody 기록

- [ ] **분석 과정 문서**
  - 분석 단계별 로그
  - 중간 결과 기록
  - 이상 사항 기록
  - **구현**: 감사 로그
